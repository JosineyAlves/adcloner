Guia Detalhado de Integração do Facebook Login

Este guia fornecerá um passo a passo detalhado para integrar o Facebook Login em seu aplicativo web usando o SDK JavaScript, incluindo exemplos de código.

1. Antes de Começar

Para integrar o Facebook Login, você precisará do seguinte:

  • Um ID de Aplicativo do Facebook para um site.
  • Uma conta de Desenvolvedor do Facebook.

2. Configuração Básica e SDK JavaScript

O primeiro passo é adicionar o SDK JavaScript do Facebook à sua página web e inicializá-lo com seu ID de aplicativo e a versão da API Graph.

Adicionar o SDK JavaScript

Inclua o seguinte código HTML no corpo da sua página, preferencialmente logo após a tag <body> de abertura:

<!DOCTYPE html>
<html lang="en">
  <head></head>
  <body>

    <h2>Adicionar Facebook Login à sua página web</h2>

    <p id="profile"></p>

    <script>
      // Adicionar o Facebook SDK para Javascript
      (function(d, s, id){
        var js, fjs = d.getElementsByTagName(s)[0];
        if (d.getElementById(id)) {return;}
        js = d.createElement(s); js.id = id;
        js.src = "https://connect.facebook.net/en_US/sdk.js";
        fjs.parentNode.insertBefore(js, fjs);
      }(document, 'script', 'facebook-jssdk'));

      window.fbAsyncInit = function() {
        // Inicializar o SDK com seu aplicativo e a versão da API Graph
        FB.init({
          appId            : '{seu-id-de-aplicativo-facebook}',
          xfbml            : true,
          version          : '{a-versao-da-api-graph-para-seu-aplicativo}'
        });

        // Se você estiver logado, obtenha automaticamente seu nome e endereço de e-mail
        FB.login(function(response) {
          if (response.authResponse) {
            console.log('Bem-vindo! Buscando suas informações.... ');
            FB.api('/me', {fields: 'name, email'}, function(response) {
              document.getElementById("profile").innerHTML = "Que bom te ver, " + response.name + ". Seu endereço de e-mail é " + response.email;
            });
          } else {
            // Se você não estiver logado, a caixa de diálogo de login será aberta para você fazer login
            console.log('Usuário cancelou o login ou não autorizou totalmente.');
          }
        });
      };
    </script>

  </body>
</html>

Observações:

  • Substitua {seu-id-de-aplicativo-facebook} pelo ID real do seu aplicativo.
  • Substitua {a-versao-da-api-graph-para-seu-aplicativo} pela versão da API Graph que você deseja usar (ex: v19.0).
  • As permissões public_profile e email não exigem revisão do aplicativo e são concedidas automaticamente a todos os aplicativos que usam o Facebook Login.

Habilitar SDK JavaScript para Facebook Login no Painel do Aplicativo

No Painel do Aplicativo, escolha seu aplicativo e navegue até Adicionar um Produto. Clique em Configurar no cartão Facebook Login. Selecione Configurações no painel de navegação esquerdo e, em Client OAuth Settings, insira sua URL de redirecionamento no campo Valid OAuth Redirect URIs para autorização bem-sucedida.

Indique que você está usando o SDK JavaScript para login definindo a opção Login with JavaScript SDK como “sim”, e insira o domínio da sua página que hospeda o SDK na lista Allowed Domains for JavaScript SDK. Isso garante que os tokens de acesso sejam retornados apenas para callbacks em domínios autorizados. Apenas páginas HTTPS são suportadas para ações de autenticação com o SDK JavaScript do Facebook.

3. Verificar o Status de Login de uma Pessoa

Ao carregar sua página web, é importante determinar se uma pessoa já está logada em sua página com o Facebook Login. Uma chamada para FB.getLoginStatus() inicia uma chamada para o Facebook para obter o status de login. O Facebook então chama sua função de callback com os resultados.

Exemplo de Chamada

FB.getLoginStatus(function(response) {
    statusChangeCallback(response);
});

function statusChangeCallback(response) {
  console.log(response);
  // A resposta inclui o status de login e o authResponse, se conectado
  if (response.status === 'connected') {
    // Logado em sua página web e no Facebook.
    console.log('Usuário conectado.');
    // Você pode agora chamar a API Graph para obter informações do usuário
    FB.api('/me', {fields: 'name,email'}, function(response) {
      console.log('Informações do usuário: ' + JSON.stringify(response));
    });
  } else if (response.status === 'not_authorized') {
    // Logado no Facebook, mas não autorizou seu aplicativo.
    console.log('Usuário não autorizou o aplicativo.');
  } else {
    // Não logado no Facebook, ou não é possível determinar.
    console.log('Usuário não logado no Facebook.');
  }
}

Exemplo de Resposta JSON

{
    "status": "connected",
    "authResponse": {
        "accessToken": "{access-token}",
        "expiresIn": "{unix-timestamp}",
        "reauthorize_required_in": "{seconds-until-token-expires}",
        "signedRequest": "{signed-parameter}",
        "userID": "{user-id}"
    }
}

O campo status especifica o status de login da pessoa usando a página web. Os possíveis valores são:

  • connected: A pessoa está logada no Facebook e logou em sua página web.
  • not_authorized: A pessoa está logada no Facebook, mas não logou em sua página web.
  • unknown: A pessoa não está logada no Facebook, então você não sabe se ela logou em sua página web. Ou FB.logout() foi chamado antes, e, portanto, não é possível conectar ao Facebook.

Se o status for connected, os seguintes parâmetros authResponse são incluídos na resposta:

  • accessToken: Um token de acesso para a pessoa usando a página web.
  • expiresIn: Um timestamp UNIX quando o token expira. Uma vez que o token expira, a pessoa precisará fazer login novamente.
  • reauthorize_required_in: A quantidade de tempo antes que o login expire, em segundos, e a pessoa precisará fazer login novamente.
  • signedRequest: Um parâmetro assinado que contém informações sobre a pessoa usando sua página web.
  • userID: O ID da pessoa usando sua página web.

O SDK JavaScript detecta automaticamente o status de login, então você não precisa fazer nada para habilitar esse comportamento.

4. Fazer Login de uma Pessoa

Se uma pessoa abrir sua página web, mas não estiver logada ou não estiver logada no Facebook, você pode usar a caixa de diálogo de Login para solicitar que ela faça login em ambos. Se ela não estiver logada no Facebook, será solicitado primeiro que ela faça login no Facebook, e depois que faça login em sua página web.

Existem duas maneiras de fazer login de alguém:

  • O Botão de Login do Facebook
  • A Caixa de Diálogo de Login do SDK JavaScript

A. Usando o Botão de Login do Facebook

Para usar o Botão de Login do Facebook, utilize o Configurador de Plugin para personalizar o botão de login e obter o código HTML. O configurador permite definir largura, tamanho do botão, texto, layout, e se deve incluir o nome e a foto do perfil quando o usuário estiver logado.

B. Fazer Login com a Caixa de Diálogo de Login do SDK JavaScript

Para usar seu próprio botão de login, invoque a Caixa de Diálogo de Login com uma chamada para FB.login().

FB.login(function(response){
  // handle the response
});

Solicitar Permissões Adicionais

Quando uma pessoa clica em seu botão HTML, uma janela pop-up com a caixa de diálogo de Login é exibida. Esta caixa de diálogo permite que você solicite permissão para acessar os dados de uma pessoa. O parâmetro scope pode ser passado junto com a chamada da função FB.login(). Este parâmetro opcional é uma lista de permissões, separadas por vírgulas, que uma pessoa deve confirmar para dar à sua página web acesso aos seus dados. O Facebook Login requer a permissão public_profile avançada, a ser usada por usuários externos.

Exemplo de Chamada com Permissões

Este exemplo solicita à pessoa que está fazendo login se sua página web pode ter permissão para acessar seu perfil público e e-mail.

FB.login(function(response) {
  // handle the response
}, {scope: 'public_profile,email'});

Lidar com a Resposta da Caixa de Diálogo de Login

A resposta, seja para conectar ou cancelar, retorna um objeto authResponse para o callback especificado quando você chamou FB.login(). Esta resposta pode ser detectada e tratada dentro da função FB.login().

Exemplo de Tratamento de Resposta

FB.login(function(response) {
  if (response.status === 'connected') {
    // Logado em sua página web e no Facebook.
    console.log('Login bem-sucedido!');
    // Você pode agora redirecionar o usuário ou carregar conteúdo personalizado
  } else {
    // A pessoa não está logada em sua página web ou não é possível determinar.
    console.log('Login cancelado ou falhou.');
  }
});

5. Fazer Logout de uma Pessoa

Para fazer logout de uma pessoa da sua página web, você pode anexar a função FB.logout() do SDK JavaScript a um botão ou a um evento.

Exemplo de Logout

<a href="#" onclick="FB.logout(function(response) { alert('Você foi desconectado.'); });">Sair do Facebook</a>
FB.logout(function(response) {
  // A pessoa agora está desconectada do seu aplicativo e do Facebook.
  console.log('Usuário desconectado.');
});
